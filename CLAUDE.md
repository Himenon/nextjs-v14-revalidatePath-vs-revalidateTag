# Claude 作業ガイド

## コミットのルール

変更のたびに `git commit` を行うこと。コミットメッセージは lefthook が自動生成するため、メッセージを指定せずにコミットする。

```bash
git add <変更ファイル>
git commit
```

## 禁忌

- `tsconfig.json` の`paths`を使わないでください。使ったら出直してこい。
- `正しい`という言葉を使うな。
- 抽象的な言葉を使うな。ソフトウェアは具体的な事象を作る行為である。
- 型定義の適用されていない変数宣言をするな。定義したら出直してこい。
- アプリケーション全体の設定を調整してはいけない。
